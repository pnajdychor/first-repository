# Zadanie: GitHub Actions -- Pobieranie informacji o pogodzie

## (PL)

Twoim zadaniem jest przygotowanie workflow GitHub Actions, który
umożliwia ręczne wywołanie procesu pobierania informacji o pogodzie dla
wybranego miasta. Workflow powinien:

### Wymagania:

-   Być uruchamiany ręcznie (`workflow_dispatch`).
-   Przyjmować parametr `city` typu `string`, z domyślną wartością
    `Warsaw`.
-   Używać zmiennej środowiskowej `MY_KLUCZYK`, reprezentującej klucz
    API do usługi pogodowej.
-   Wykonywać job o nazwie **Pobierz informacje o pogodzie**, działający
    na `ubuntu-latest`.
-   Domyślnie wykonywać komendy w katalogu:
    `./CWICZENIA/06-context/pogoda`.
-   W pierwszym kroku wykonywać checkout repozytorium
    (`actions/checkout@v3`).
-   Następnie uruchamiać skrypt `start.sh`, przekazując mu:
    -   nazwę miasta (`${{ inputs.city }}`)
    -   klucz API (`${{ env.MY_KLUCZYK }}`)
-   Wynik działania skryptu powinien zostać dopisany do
    `$GITHUB_STEP_SUMMARY`.

Skrypt `start.sh` jest dostarczony jako część zadania i należy założyć,
że przyjmuje dwa argumenty: `city` oraz `apiKey`.

Twoim celem jest przygotowanie kompletnego pliku YAML definiującego
workflow GitHub Actions spełniającego wszystkie powyższe wymagania.

# Task: GitHub Actions -- Retrieve Weather Information

## (EN)

Your task is to prepare a GitHub Actions workflow that allows manually
triggering a process for retrieving weather information for a selected
city. The workflow must:

### Requirements:

-   Be triggered manually using `workflow_dispatch`.
-   Accept an input parameter named `city` of type `string`, with a
    default value of `Warsaw`.
-   Define an environment variable `MY_KLUCZYK` representing the API key
    for the weather service.
-   Execute a job called **Pobierz informacje o pogodzie** running on
    `ubuntu-latest`.
-   Set default working directory to: `./CWICZENIA/06-context/pogoda`.
-   Perform repository checkout using `actions/checkout@v3`.
-   Execute the `start.sh` script, passing:
    -   the city (`${{ inputs.city }}`)
    -   the API key (`${{ env.MY_KLUCZYK }}`)
-   Append the script output to `$GITHUB_STEP_SUMMARY`.

The `start.sh` script is included as part of the task and is assumed to
accept two arguments: `city` and `apiKey`.

Your goal is to create a complete YAML file defining a GitHub Actions
workflow that fulfills all requirements listed above.
